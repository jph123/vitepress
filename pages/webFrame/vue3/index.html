Vue 3

Composition API、Proxy 响应式、性能全面提升，Vue 的全新篇章。以下是 Vue3 面试高频知识点。

高频面试题

01Vue3 相比 Vue2 有哪些主要改进？▼
  • 响应式系统：Object.defineProperty → Proxy，支持数组索引/长度监听、Map/Set
  • Composition API：逻辑组合复用，替代 mixin
  • 性能提升：编译优化（静态提升、PatchFlag）、更快的虚拟 DOM
  • Tree-shaking：按需引入 API，减小打包体积
  • Fragment：组件可以多根节点
  • Teleport：组件内容传送到 DOM 任意位置
  • Suspense：异步组件加载状态管理
  • TypeScript 支持：源码用 TS 重写，类型支持更好
02Composition API 和 Options API 的区别？▼

Options API：按选项组织代码（data/methods/computed/watch），相关逻辑分散在不同选项中。

Composition API：按功能组织代码，相关逻辑集中在一起，可提取为 composable 复用。

<script setup>
import { ref, computed, onMounted } from 'vue';

// 所有用户相关逻辑在一起
const name = ref('Tom');
const age = ref(18);
const info = computed(() => `${name.value}, ${age.value}`);

onMounted(() => {
  console.log('mounted');
});
</script>
Composition API 的核心优势：逻辑复用更清晰（composable），TypeScript 支持更好，大组件可维护性更高。
03Vue3 响应式原理（Proxy）和 Vue2 有什么区别？▼

Vue2（Object.defineProperty）：

  • 只能劫持已存在的属性，新增/删除属性需 $set
  • 无法监听数组索引和 length 变化（需重写数组方法）
  • 初始化时递归遍历，性能开销大

Vue3（Proxy）：

  • 代理整个对象，新增/删除属性自动响应
  • 支持数组索引、length、Map/Set 的监听
  • 惰性响应式：访问时才递归代理，性能更好
// Vue3 响应式简化原理
function reactive(target) {
  return new Proxy(target, {
    get(obj, key, receiver) {
      track(obj, key);  // 收集依赖
      const result = Reflect.get(obj, key, receiver);
      return typeof result === 'object' ? reactive(result) : result;
    },
    set(obj, key, value, receiver) {
      const result = Reflect.set(obj, key, value, receiver);
      trigger(obj, key);  // 触发更新
      return result;
    }
  });
}
04ref 和 reactive 的区别？▼
  • ref：用于基本类型和引用类型，通过 .value 访问，模板中自动解包
  • reactive：只用于对象/数组，直接访问属性，不需要 .value
import { ref, reactive } from 'vue';

// ref
const count = ref(0);
count.value++;          // JS 中用 .value
<span>{{ count }}</span>  // 模板中自动解包

// reactive
const state = reactive({
  name: 'Tom',
  age: 18
});
state.age = 20;         // 直接修改
<span>{{ state.age }}</span>
注意：reactive 不能直接替换整个对象（会丢失响应式），需要用 Object.assign 或 ref。解构 reactive 对象也会丢失响应式，需用 toRefs。
05watch 和 watchEffect 的区别？▼
  • watch：需要明确指定监听的数据源，可获取新旧值，默认不立即执行
  • watchEffect：自动收集回调中的响应式依赖，立即执行一次，无法获取旧值
// watch：明确指定监听
watch(count, (newVal, oldVal) => {
  console.log(`${oldVal} → ${newVal}`);
}, { immediate: true, deep: true });

// 监听多个
watch([count, name], ([newCount, newName]) => { ... });

// watchEffect：自动收集依赖
watchEffect(() => {
  console.log(count.value);  // 自动监听 count
  console.log(state.name);   // 自动监听 state.name
});
选择建议：需要旧值或精确控制监听目标用 watch；只关心副作用、不关心旧值用 watchEffect。
06说说 Vue3 的新组件 Fragment、Teleport、Suspense▼

Fragment（碎片）：组件不再需要唯一根节点，可以返回多个根节点。

<template>
  <header>Header</header>
  <main>Content</main>
  <footer>Footer</footer>
<!-- Vue2 需要包一个 div，Vue3 不需要 -->
</template>

Teleport（传送门）：将组件内容渲染到指定 DOM 节点，常用于弹窗。

<teleport to="body">
  <div class="modal">弹窗内容</div>
</teleport>
<!-- 渲染到 body 下，避免父级 overflow/z-index 问题 -->

Suspense（悬念）：管理异步组件的加载状态。

<suspense>
  <template #default>
    <AsyncComponent />
  </template>
  <template #fallback>
    <div>加载中...</div>
  </template>
</suspense>
07Vue3 的生命周期有哪些变化？▼

Vue3 生命周期改为了 setup 中调用的钩子函数，名称加了 on 前缀：

  • beforeCreate / created → 直接在 setup 中写
  • beforeMount → onBeforeMount
  • mounted → onMounted
  • beforeUpdate → onBeforeUpdate
  • updated → onUpdated
  • beforeDestroy → onBeforeUnmount（改名）
  • destroyed → onUnmounted（改名）
<script setup>
import { onMounted, onBeforeUnmount } from 'vue';

onMounted(() => {
  console.log('DOM 已挂载');
});

onBeforeUnmount(() => {
  console.log('组件即将卸载');
  // 清除定时器、事件监听等
});
</script>
08script setup 语法糖有什么好处？▼

<script setup> 是 Vue3.2+ 的编译时语法糖，简化了 Composition API 的写法：

  • 无需 return：顶层变量/函数自动暴露给模板
  • 无需 components 注册：import 的组件直接使用
  • 更好的运行时性能：编译时优化
  • 更好的 TypeScript 支持
<script setup>
import { ref } from 'vue';
import ChildComp from './Child.vue';

// 自动暴露给模板
const count = ref(0);
const increment = () => count.value++;

// defineProps / defineEmits 编译宏
const props = defineProps({ title: String });
const emit = defineEmits(['change']);
</script>

<template>
  <h1>{{ props.title }}</h1>
  <button @click="increment">{{ count }}</button>
  <ChildComp />
</template>
09Vue3 的 Tree-shaking 是怎么实现的？▼

Vue3 的全局 API 改为通过 ES Module 按需导入，打包工具可以摇掉未使用的代码：

// Vue2：所有 API 挂载在 Vue 上
import Vue from 'vue';
Vue.nextTick();  // 即使不用也打包进去

// Vue3：按需导入
import { nextTick, ref } from 'vue';
// 不导入的 API 会被 Tree-shaking 移除

效果：Vue3 的最小打包体积从 Vue2 的 ~20KB 降到 ~10KB（gzip 后）。

原理：Vue3 源码大量使用函数声明 + ES Module export，没有副作用，打包工具可以安全移除未引用的函数。
10如何编写自定义 Composable（Hook）？▼

Composable 是 Vue3 替代 mixin 的逻辑复用方案，约定以 use 开头命名：

// useMouse.js
import { ref, onMounted, onUnmounted } from 'vue';

export function useMouse() {
  const x = ref(0);
  const y = ref(0);

  const update = (e) => {
    x.value = e.pageX;
    y.value = e.pageY;
  };

  onMounted(() => {
    window.addEventListener('mousemove', update);
  });

  onUnmounted(() => {
    window.removeEventListener('mousemove', update);
  });

  return { x, y };
}

// 使用
<script setup>
import { useMouse } from './useMouse';
const { x, y } = useMouse();
</script>

<template>Mouse: {{ x }}, {{ y }}</template>
优势：相比 mixin，来源清晰、无命名冲突、TypeScript 类型推导完整。
