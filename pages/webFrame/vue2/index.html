Vue 2

渐进式 JavaScript 框架，Vue 2 经典版本。以下是 Vue2 面试高频知识点。

高频面试题

01Vue2 的生命周期有哪些？▼

Vue2 生命周期分为四个阶段：

  • 创建阶段：beforeCreate → created（可访问 data/methods）
  • 挂载阶段：beforeMount → mounted（DOM 已渲染）
  • 更新阶段：beforeUpdate → updated
  • 销毁阶段：beforeDestroy → destroyed
常见面试追问：在哪发请求？→ created 或 mounted。在哪操作 DOM？→ mounted。在哪清除定时器？→ beforeDestroy。
export default {
  data() { return { msg: 'hello' } },
  created() {
    console.log('data 已初始化', this.msg);
  },
  mounted() {
    console.log('DOM 已挂载');
    this.timer = setInterval(() => {}, 1000);
  },
  beforeDestroy() {
    clearInterval(this.timer); // 清除定时器
  }
}
02Vue2 双向绑定原理是什么？▼

Vue2 基于 Object.defineProperty 实现数据劫持 + 发布订阅模式：

  1. Observer：递归遍历 data，用 defineProperty 的 getter/setter 劫持每个属性
  2. Dep：每个属性都有一个依赖收集器，getter 时收集 Watcher
  3. Watcher：setter 触发时通知所有 Watcher 更新视图
  4. Compiler：解析模板指令，初始化视图并绑定更新函数
// 简化版原理
Object.defineProperty(obj, 'key', {
  get() {
    Dep.target && dep.addDep(Dep.target); // 收集依赖
    return value;
  },
  set(newVal) {
    if (newVal === value) return;
    value = newVal;
    dep.notify(); // 通知更新
  }
});
局限性：无法监听数组下标修改和对象属性新增/删除（需用 $set / $delete）。Vue3 改用 Proxy 解决了此问题。
03Vue 组件间通信有哪些方式？▼
  • 父→子：props
  • 子→父：$emit 触发自定义事件
  • 兄弟组件：EventBus（$on / $emit）、Vuex
  • 跨层级：provide / inject（祖先→后代）
  • 任意组件：Vuex / Pinia 状态管理
  • ref：父组件直接访问子组件实例
  • $attrs / $listeners：透传属性和事件
// 父子通信
// 父组件
<Child :msg="message" @update="handleUpdate" />

// 子组件
export default {
  props: ['msg'],
  methods: {
    notify() { this.$emit('update', 'new value'); }
  }
}
04computed 和 watch 的区别？▼
  • computed：计算属性，有缓存，依赖不变时不重新计算，必须返回值
  • watch：监听器，数据变化时执行副作用（异步操作、深度监听），不需要返回值
export default {
  data() {
    return { firstName: 'Peng', lastName: 'Hui' };
  },
  computed: {
    fullName() {
      // 依赖 firstName/lastName，有缓存
      return this.firstName + ' ' + this.lastName;
    }
  },
  watch: {
    fullName(newVal, oldVal) {
      // fullName 变化时执行副作用
      console.log('名字变了:', newVal);
    }
  }
}
能用 computed 就不用 watch。watch 适合异步或开销较大的操作。
05v-if 和 v-show 的区别？▼
  • v-if：条件为 false 时不渲染 DOM，切换时创建/销毁元素，有更高的切换开销
  • v-show：始终渲染 DOM，通过 display: none 控制显隐，有更高的初始渲染开销
使用场景：频繁切换用 v-show，条件很少改变用 v-if。v-if 可配合 v-else / v-else-if 使用。

注意：v-if 和 v-for 不要一起用！v-for 优先级更高，导致每次循环都判断 v-if。应该用 computed 过滤数据。

06v-for 中 key 的作用是什么？▼

key 是虚拟 DOM Diff 算法的节点标识，帮助 Vue 识别哪些元素改变了：

  • 高效更新虚拟 DOM：复用已有节点，只移动而非重新创建
  • 避免就地复用导致的渲染错误（如输入框内容错位）
<!-- 正确：用唯一 id -->
<li v-for="item in list" :key="item.id">{{ item.name }}</li>

<!-- 错误：用 index 作为 key -->
<li v-for="(item, index) in list" :key="index">{{ item.name }}</li>
用 index 做 key 的问题：列表插入/删除时，index 变化导致 Diff 复用错误节点，可能引起数据错乱和性能下降。
07$nextTick 的作用是什么？▼

Vue 更新 DOM 是异步的（批量更新策略）。$nextTick 在下次 DOM 更新循环结束后执行回调，确保能拿到更新后的 DOM。

export default {
  methods: {
    updateMsg() {
      this.msg = '新内容';
      console.log(this.$el.textContent); // 旧内容
      this.$nextTick(() => {
        console.log(this.$el.textContent); // 新内容
      });
    }
  }
}
原理：内部使用微任务（Promise.then / MutationObserver），确保在 DOM 更新后执行。
08说说 Vue 的虚拟 DOM 和 Diff 算法▼

虚拟 DOM：用 JS 对象描述真实 DOM 结构，数据变化先生成新 VNode，与旧 VNode 对比（Diff），最后只更新差异部分。

Diff 算法策略（同层比较）：

  • 同层比较：只比较同一层级的节点
  • 类型相同：复用 DOM，对比属性差异
  • 类型不同：直接替换整个子树
  • 列表 Diff：用 key 标识，尽量复用和移动节点，减少创建/删除
// 虚拟 DOM 结构
{
  tag: 'div',
  props: { class: 'container' },
  children: [
    { tag: 'p', props: {}, children: ['Hello'] }
  ]
}
09keep-alive 的作用是什么？▼

<keep-alive> 缓存不活动的组件实例，避免重复渲染，保留组件状态。

<keep-alive include="UserList,Detail" 
            exclude="Login"
            :max="10">
  <router-view />
</keep-alive>
  • include / exclude：指定缓存哪些组件
  • max：最大缓存数量（LRU 淘汰策略）

独有的生命周期：

  • activated：组件被激活时调用
  • deactivated：组件被缓存时调用
10mixin 的作用和问题？▼

mixin 用于抽取组件间的公共逻辑，实现代码复用。

// mixin.js
export const myMixin = {
  data() { return { count: 0 } },
  methods: {
    increment() { this.count++; }
  }
};

// 组件中使用
export default {
  mixins: [myMixin],
  // 自动合并 data/methods/生命周期
}

mixin 的问题：

  • 命名冲突：多个 mixin 的属性/方法可能重名
  • 数据来源不清晰：组件中不知道某个方法来自哪个 mixin
  • 调试困难：错误堆栈难以追踪
Vue3 用 Composition API（composables）替代 mixin，解决了以上问题。
