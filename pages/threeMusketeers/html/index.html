HTML

超文本标记语言（HyperText Markup Language），构建网页结构的基石。以下是面试中高频出现的 HTML 知识点。

高频面试题

01DOCTYPE 的作用是什么？▼

<!DOCTYPE html> 声明位于文档最前面，用于告诉浏览器以什么标准解析页面。

  • HTML5：只需写 <!DOCTYPE html>，触发标准模式
  • 不写 DOCTYPE：浏览器进入怪异模式（Quirks Mode），盒模型、对齐方式等行为与标准不一致
面试要点：DOCTYPE 不是 HTML 标签，它是一条指令，告诉浏览器使用哪种 HTML 规范来渲染页面。
02说说 HTML5 的新特性▼

HTML5 相比 HTML4 增加了以下重要特性：

  • 语义化标签：header、nav、section、article、aside、footer
  • 多媒体标签：audio、video 取代 Flash
  • Canvas / SVG：原生绘图能力
  • 表单增强：date、email、number、range、color 等新 input 类型
  • 本地存储：localStorage、sessionStorage、IndexedDB
  • 地理位置：Geolocation API
  • 拖拽 API：Drag and Drop
  • Web Worker：多线程 JavaScript
  • WebSocket：全双工通信
03行内元素、块级元素和行内块元素的区别？▼
  • 块级元素：独占一行，可设宽高。div、p、h1~h6、ul、table
  • 行内元素：不独占一行，宽高由内容决定，不可设宽高。span、a、strong、em、label
  • 行内块元素：不独占一行，可设宽高。img、input、button、td
可通过 display 属性相互转换：display:block / inline / inline-block
04localStorage、sessionStorage、cookie 的区别？▼
  • cookie：大小 4KB，每次请求自动携带，可设过期时间，主要用于身份认证
  • localStorage：大小 5~10MB，永久存储，手动清除才消失，同源共享
  • sessionStorage：大小 5~10MB，当前会话有效，关闭标签页即清除
// localStorage 基本操作
localStorage.setItem('key', 'value');
localStorage.getItem('key');
localStorage.removeItem('key');
localStorage.clear();
05什么是语义化标签？为什么要语义化？▼

语义化标签是指用有意义的标签名来描述内容，如 <header> 表示页头，<nav> 表示导航，<article> 表示独立文章。

语义化的好处：

  • 结构清晰，代码可读性更好
  • 利于 SEO，搜索引擎更容易理解页面内容
  • 方便屏幕阅读器等辅助设备解析，提升无障碍访问
  • 在没有 CSS 的情况下页面也能呈现良好的结构
06script 标签的 defer 和 async 属性有什么区别？▼
  • 无属性：遇到 script 立即下载并执行，阻塞 HTML 解析
  • async：异步下载，下载完成后立即执行，执行时阻塞解析。多个 async 脚本执行顺序不确定
  • defer：异步下载，等到 HTML 解析完成后按顺序执行（在 DOMContentLoaded 之前）
<!-- 不阻塞，按顺序执行，推荐 -->
<script src="app.js" defer></script>

<!-- 不阻塞，下载完就执行，适合独立脚本 -->
<script src="analytics.js" async></script>
面试结论：依赖 DOM 的脚本用 defer，不依赖 DOM 的独立脚本用 async。
07从输入 URL 到页面展示发生了什么？▼
  1. DNS 解析：域名 → IP 地址（递归查询 + 缓存）
  2. TCP 连接：三次握手建立连接（HTTPS 还有 TLS 握手）
  3. 发送 HTTP 请求
  4. 服务器处理：返回响应（HTML / JSON 等）
  5. 浏览器解析渲染：
    • 解析 HTML 构建 DOM 树
    • 解析 CSS 构建 CSSOM 树
    • DOM + CSSOM 合并成 渲染树（Render Tree）
    • 布局（Layout）：计算元素位置大小
    • 绘制（Paint）：填充像素
    • 合成（Composite）：GPU 合成图层
08如何优化页面加载性能？▼
  • 减少 HTTP 请求：合并文件、雪碧图、内联小资源
  • 压缩资源：Gzip / Brotli、压缩图片（WebP）、压缩 JS/CSS
  • 缓存策略：强缓存（Cache-Control）、协商缓存（ETag）
  • 懒加载：图片 loading="lazy"、组件按需加载
  • CDN 加速：静态资源分发到边缘节点
  • 预加载：<link rel="preload"> 提前加载关键资源
  • 减少回流重绘：批量操作 DOM、使用 transform 代替 top/left
  • 合理使用 defer/async：避免脚本阻塞渲染
09meta viewport 的作用是什么？▼

控制移动端页面的视口大小和缩放，是响应式设计的必备标签：

<meta name="viewport" 
      content="width=device-width, initial-scale=1, 
      maximum-scale=1, user-scalable=no">
  • width=device-width：视口宽度等于设备宽度
  • initial-scale=1：初始缩放比例为 1
  • user-scalable=no：禁止用户手动缩放
不写这个标签，移动端浏览器会以默认 980px 宽度渲染，导致页面缩放变小。
10iframe 有哪些优缺点？▼

优点：

  • 隔离样式和脚本，适合嵌入第三方内容
  • 并行加载，不阻塞主页面

缺点：

  • 不利于 SEO，搜索引擎爬虫不抓取 iframe 内容
  • 阻塞主页面的 onload 事件
  • 增加 HTTP 请求，影响性能
  • 跨域通信复杂（需用 postMessage）
  • 移动端体验差，滚动和缩放有问题
