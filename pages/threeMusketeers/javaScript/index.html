JavaScript

网页交互的灵魂，前端必备编程语言。以下是 JavaScript 面试高频知识点，也是进阶前端的必修课。

高频面试题

01什么是闭包？闭包的应用场景有哪些？▼

闭包是指一个函数能够访问其外部函数作用域中的变量，即使外部函数已经执行完毕。

function createCounter() {
  let count = 0;  // 外部变量
  return function() {  // 闭包
    count++;
    return count;
  };
}
const counter = createCounter();
counter(); // 1
counter(); // 2

应用场景：

  • 数据私有化（模块模式）
  • 函数柯里化（Currying）
  • 防抖节流函数实现
  • 回调函数中保持变量引用
注意：闭包会一直占用内存，不当使用可能导致内存泄漏。
02说说原型与原型链▼
  • 每个对象都有一个 __proto__（隐式原型），指向其构造函数的 prototype
  • 每个函数都有一个 prototype（显式原型），指向一个原型对象
  • 原型对象上有 constructor 指回构造函数本身
function Person(name) { this.name = name; }
Person.prototype.sayHi = function() {
  console.log('Hi, ' + this.name);
};

const p = new Person('Tom');
p.sayHi(); // Hi, Tom

// 原型链：
// p.__proto__ → Person.prototype
// Person.prototype.__proto__ → Object.prototype
// Object.prototype.__proto__ → null

查找过程：访问对象属性时，先在自身找，找不到沿 __proto__ 向上查找，直到 null，这条链就是原型链。

03this 的指向规则是什么？▼
  1. 默认绑定：独立函数调用，this → window（严格模式 undefined）
  2. 隐式绑定：作为对象方法调用，this → 调用的对象
  3. 显式绑定：call / apply / bind，this → 指定的对象
  4. new 绑定：this → 新创建的对象
  5. 箭头函数：没有自己的 this，继承外层作用域的 this
// 优先级：new > 显式 > 隐式 > 默认
const obj = {
  name: 'Tom',
  fn: () => console.log(this.name)  // 箭头函数，this 继承外层
};
obj.fn(); // undefined（不是 'Tom'）

const obj2 = {
  name: 'Jerry',
  fn() { console.log(this.name); }
};
obj2.fn(); // Jerry（隐式绑定）
04说说 JavaScript 的事件循环（Event Loop）▼

JavaScript 是单线程的，通过事件循环实现异步非阻塞：

  • 调用栈（Call Stack）：同步代码依次执行
  • 宏任务队列（Macrotask）：setTimeout、setInterval、I/O、UI 渲染
  • 微任务队列（Microtask）：Promise.then、MutationObserver、queueMicrotask
console.log('1');              // 同步

setTimeout(() => {
  console.log('2');            // 宏任务
}, 0);

Promise.resolve().then(() => {
  console.log('3');            // 微任务
});

console.log('4');              // 同步

// 输出顺序: 1 → 4 → 3 → 2
执行顺序：同步代码 → 清空微任务队列 → 执行一个宏任务 → 清空微任务 → 循环
05Promise 是什么？async/await 的原理？▼

Promise 是异步操作的解决方案，有三种状态：pending → fulfilled / rejected，状态一旦改变不可逆。

// Promise 链式调用
fetch('/api')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await 是 Promise 的语法糖
async function getData() {
  try {
    const res = await fetch('/api');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

常用 API：Promise.all()（全部成功）、Promise.race()（最快返回）、Promise.allSettled()（等所有完成）

06深拷贝和浅拷贝的区别？如何实现？▼
  • 浅拷贝：只拷贝第一层，嵌套对象仍共享引用
  • 深拷贝：递归拷贝所有层级，完全独立
// 浅拷贝
const shallow = { ...obj };           // 展开运算符
const shallow = Object.assign({}, obj);

// 深拷贝
const deep = structuredClone(obj);     // 现代方案（推荐）
const deep = JSON.parse(JSON.stringify(obj)); // 简单方案，不支持函数/undefined/循环引用

// 手写深拷贝
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (map.has(obj)) return map.get(obj);
  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key], map);
    }
  }
  return clone;
}
07防抖和节流的区别？如何实现？▼
  • 防抖（Debounce）：事件触发后等待一段时间再执行，期间再次触发则重新计时。只执行最后一次
  • 节流（Throttle）：规定时间内只执行一次。降低执行频率
// 防抖：搜索框输入
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流：滚动事件
function throttle(fn, interval) {
  let last = 0;
  return function(...args) {
    const now = Date.now();
    if (now - last >= interval) {
      last = now;
      fn.apply(this, args);
    }
  };
}
防抖适用于搜索输入、窗口 resize；节流适用于滚动加载、按钮防连点。
08var、let、const 的区别？▼
  • var：函数作用域，有变量提升，可重复声明，值可变
  • let：块级作用域，有暂时性死区，不可重复声明，值可变
  • const：块级作用域，必须初始化，不可重新赋值（但对象内部属性可改）
console.log(a); // undefined（变量提升）
var a = 1;

console.log(b); // ReferenceError（暂时性死区）
let b = 2;

const c = 3;
c = 4;          // TypeError
const obj = {};
obj.name = 'Tom'; // OK，修改属性不是重新赋值
09箭头函数和普通函数有什么区别？▼
  • 没有自己的 this，继承外层作用域的 this
  • 没有 arguments 对象（用 ...args 代替）
  • 不能作为构造函数，不能使用 new
  • 没有 prototype 属性
  • 不能用作 Generator 函数
// 箭头函数的 this 继承外层
const obj = {
  name: 'Tom',
  // 普通函数：this 指向 obj
  greet() { console.log(this.name); },
  // 箭头函数：this 继承外层（window/undefined）
  arrow: () => console.log(this.name)
};
obj.greet();  // Tom
obj.arrow();  // undefined
10说说 JavaScript 的垃圾回收机制▼

JS 有自动垃圾回收机制，常见算法：

  • 标记清除（Mark-Sweep）：从根对象出发，标记所有可达对象，清除未标记的。现代 V8 引擎主要使用此算法的改进版
  • 引用计数（Reference Counting）：引用数为 0 时回收。但无法处理循环引用

常见内存泄漏场景：

  • 意外的全局变量（未用 let/const 声明）
  • 被遗忘的定时器（setInterval 未清除）
  • 闭包引用不需要的变量
  • 脱离 DOM 的引用（已删除的元素仍被变量引用）
  • 事件监听器未移除
// 内存泄漏示例
function leak() {
  const bigData = new Array(1000000);
  return function() {
    console.log('leak'); // bigData 被闭包引用，无法回收
  };
}
